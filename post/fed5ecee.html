算法导论笔记(2):渐进记号及递归式渐进界求解方法介绍(代入法,递归树法和主方法)

发表于 2017-09-22 | 分类于 算法导论学习笔记 | 阅读次数:

算法的运行时间通常由渐进记号来刻画，本文将介绍几类渐进记号的概念，和递归式运行时间的计算方法。

渐进符号介绍

$\Theta$记号：$\Theta \left( g\left( n\right) \right) =${$f\left( n\right)$: 存在正常量$c_{1}$、$c_{2}$和$n_{0}$，使得对所有$n\geq n_{0}$,有$0\leq c_{1}g\left(n\right)\leq f\left( n\right)\leq c_{2}g\left(n\right)$}

$O$记号：$O \left( g\left( n\right) \right) =${$f\left( n\right)$:存在正常量c和$n_{0}$,使得对所有$n\geq n_{0}$,有$0\leq f\left( n\right) \leq cg\left( n\right)$}

$\Omega$记号：$\Omega \left( g\left( n\right) \right) =${$f\left( n\right)$:存在正常量c和$n_{0}$,使得对所有$n\geq n_{0}$,有$0\leq cg\left( n\right) \leq f\left( n\right)$}

直观的来看，三种渐进记号与于原函数的关系如下图：

可以看到，对于$\theta$记号，总存在正常数$c_{1}$、$c_{2}$，使得对于足够大的n，函数$f\left( n\right)$能够“夹入”$c_{1}g\left(n\right)$和$c_{2}g\left(n\right)$之间；对于O记号，$n_{0}$及其右边的所有值n，函数$f\left( n\right)$的值总小于或等于$cg\left(n\right)$；对于$\Omega$记号，在$n_{0}$及其右边的所有值n，函数$f\left( n\right)$的值总大于或等于$cg\left(n\right)$。

求解递归方程

代入法

这是一种最一般的方法，包含以下步骤：

  1. 猜测结果的形式；
  2. 用归纳法验证；
  3. 求解。

例 $T\left( n\right) =4T\left( n / 2\right) +n$ （假设$T\left( 1\right) =\Theta \left( 1\right)$）

第一步：
(1) 猜测为$O\left( n^{3}\right)$;
(2)假设对于$k < n$,有$T\left( k\right) \leq ck^{3}$;
(3)用递归方法证明$T\left( n\right) \leq cn^{3}$。

当$\left( c / 2\right) n^{3}-n\geq 0$时，不等式成立，例如$c\geq 2$且$n\geq 1$。
初始条件：令$T\left( 1\right) =\Theta \left( 1\right)$，对于所有$n< n_{0}$成立，其中$n_{0}$是一个适当的常数。显然有对于所有$1\leq n\leq n{0}$，当c足够大，有$\Theta \left( 1\right) \leq cn^{3}$。显然约束是不够紧的。

递归树法

主方法

定理（主定理)：令$a\geq1$和$b>1)是常数，f(n)是一个常数，，T(n)是定义在非负整数上的递归式：

$$ T\left( n\right) =aT\left( n / b\right) +f\left( n\right) $$

其中我们将n/b解释为$\left\lfloor n / b\right\rfloor$或$\left\lceil n / b\right\rceil$,那么T(n)有如下渐进界：

  1. 若对某个常数$\varepsilon >0$有$f\left( n\right)=O\left( n^{\log _{b}a-\varepsilon}\right)$,则$T\left( n\right)=\Theta\left( n^{\log_{b}a-\varepsilon}\right)$;
  2. 若$f\left( n\right)=\Theta\left( n^{\log _{b}a-\varepsilon}\right)$，则$T\left( n\right)=\Theta\left( n^{\log _{b}a-\varepsilon}\lg n\right)$;
  3. 若对某个常数$\varepsilon<0$有$f\left( n\right)=\Omega\left( n^{\log _{b}a+\varepsilon}\right)$,且对某个常数$c<1$和所以足够大的n有$af\left( n / b\right) \leq Cf\left( n\right)$,则$T\left( n\right) =\Theta \left( f\left( n\right) \right)$。

细节：这第一种情况中，不是f(n)小于$n^{\log_{b}a}$就够了，而是要多项式意义上的小于,即，f(n)必须渐进小于$n^{\log_{b}a}$，要相差一个$n^{\varepsilon }$,其中$\varepsilon$是大于0的常数，否则不能用主方法计算。情况三同理。

多项式意义的大于（小于）
我们都知道，针对个多项式的时间复杂度，都是以其最高阶的那项作为计算依据，例如，$O\left(n^{3}+n^{2}\right)$的复杂度也就是$O\left(n^{3}\right)$的复杂度。
多项式意义上的大于就要求必须在指数级上大于，例如一个$O\left( 0.01n^{3}\right)$程序的效率比$O\left(100n^{2}\right)$的效率低，尽管在n很小的时候，前者优于后者，然而前者在指数级上大于后者，后者时间随数据规模增长得慢，最终$O\left(n^{3}\right)$的复杂度将远远超过$O\left( n^{2}\right)$。这就可以说$O\left( 0.01n^{3}\right)$在多项式意义上大于$O\left(100n^{2}\right)$
