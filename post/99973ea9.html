算法导论笔记(6):动态规划之最长公共子序列问题

发表于 2017-11-14 | 分类于 算法导论学习笔记 | 阅读次数:

动态规划如之前介绍的分治策略一样，是一种算法设计方法，它常应用于子问题重叠的情况，即不同的子问题具有公共的子子问题。动态规划算法对每个子问题只求解一次，将其解保存在一个表格中，从而无需梅西求解一个个子子问题时都重新计算。动态规划方法通常用于求解最优化问题（optimization problem）,这类问题可以有很多个解，每个解都有一个值，我们希望寻找其中的一个最优解（注意，最优解可能不唯一）。本文通过最长公共子序列（Longest Comment Subsequence, LCS）问题为例来对动态规划方法深入的认识。

问题描述

给定两个序列\(x[1 .. m]\)和\(y[1 .. n]\)，找出x和y的最长公共子序列。

例

要注意的是最优子序列并非只有一个，BCAB、BDAB等同样也是最优子序列。

最容易让人想到，也最蛮力的一种解决办法是：检查\(x[1 .. m]\)的每个子序列，看是否也为y[1..n]的子序列。

此时，查找每个子序列需要花费\(O(n)\)的时间，而x序列包含\(2^{m}\)（可以将子序列想象成有0、1组成的序列，当x中包含该字符是记为1，否则为0，很容易得出子序列的个数为\(2^{m}\)）个子序列。这时，整个问题的最坏情况的运行时间为\(O(n2^{m})\)！是一个指数时间，是十分低效的，这显然不是我们想要的结果！

理论证明

用|s|表示长度为s的序列，令\(c[i,j]=|LCS(x[1 .. i],y[1 .. j])\)，则\(c[m,n]=|LCS(x,y)|\)

理论

证明：当\(x[i]=x[j]\)：

当\(c[i, j]=k\)，令最长公共子序列\(z[1 .. k]=LCS(x[1 .. i],y[1 .. j])\)。

则，\(z[k]=x[i](=y[j])\) ，也就是说，如果z不包含最后的这个字符，就可以向z中添加x[i]这个元素，使z变长。

所以\(z[1 .. k-1]\)是\(x[1 .. i-1]\)和S\(y[1.. j-1]\)的公共子序列(CS)。下面证明\(z[1 .. k-1]\)是\(x[1 .. i-1]\)和\(y[1.. j-1]\)的最长公共子序列(LCS)： 命题：\(z[1 .. k-1]=LCS(x[1 .. i-1], y[1 .. j-1])\) 假设w是\(x[1 .. i-1]\)和\(y[1.. j-1]\)中更长的CS，则\(\left| w\right| >k-1\) “剪贴”法：\(w||z\left[ k\right]\)（将w和z[k]连接）是\(x[1 .. i-1]\)和\(y[1.. j-1]\)的CS，且\(w||z\left[ k\right]>k\) 这与我们假设(\(c[i, j]=k\))向矛盾，所以假设不成立，原命题成立。 即\(c[i-1,j-2]=k-1\)，这表明\(c[i,j]=c[i-1,j-1]+1\)。 其它情况同理可证。

下面要介绍动态规划的第一个特征：

动态规划特征#1

最优子结构：一个问题（实例）的最优解包含了子问题的最优解。

在LCS问题中可以描述为：如果\(z=LCS(x,y)\)，那么z任何的前缀都是某个x的前缀和某个y的前缀的LCS。
