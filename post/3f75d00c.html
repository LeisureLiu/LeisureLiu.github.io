计算机硬件系统概述

发表于 2017-10-18 | 分类于 Liunx学习笔记 | 阅读次数:

最近正在读《鸟哥的Liunx私房菜》，打算做一个该书的系类的笔记，与大家分享。本书第一章介绍了计算机的基础知识，作为一个小白菜，感觉还是受益匪浅~ ps:书的版本较早，是2009年出版的，因此有的内容已经过时，不过并不影响学习，我也暂时还没有对最新的主流PC进行了解，因此就按原书记录吧。

1. 计算机硬件系统

计算机硬件组成如下图所示，其中CPU由算数逻辑单元和控制单元组成，前者主要负责程序运算和逻辑判断，后者负责协调周边组件与各单元间的工作。图中“系统单元”是指计算及壳内的主要组件，而重点在于CPU和内存储器。如图中实线箭头部分，CPU发布的控制命令使数据的流进/流出，基本上所有的数据都是流经内存储器后再转出去，并且CPU要处理的数据则完全来自于内存。

主流CPU及种类

CPU是整个计算机系统最重要的部分，主流的CPU可以分为以下两类：

类别 精简指令集（RISC）系统 复杂指令集（CISC）系统
特点 微指令集较精简，每个指令运行时间很短，完成动作单纯，执行效率较好，做复杂的事要多个指令来完成 微指令集的每个小指令可以执行一些较低阶的硬件，每条指令的长度不同，执行每条质量花费的时间较长，但每条指令可以处理的工作较丰富
举例 Sun公司的APARC系列、IBM公司的Power Arvhitrcture（ 包括PowerPC）系列和ARM系列 AMD、Intel、VIA等的x86架构的CPU
应用 APARC架构的计算机常用于学术领域的大型工作站中，以及银行金融体系的主服务；PowerPC架构的应用上，Sony公司的PS3使用的就是PowerPC架构的Cell处理器；手机、PDA、导航系统、网路设备（交换器、路由器等）几乎都是ARM架构的CPU 个人计算机 (x86架构：最早的那个由Intel发展出的CPU代号为8086，后来依此架构开发出的CPU就被成为x86架构了，后来的64位个人计算机CPU被统称为x86_64架构)

2. 计算机运作流程

鸟哥将它与一个人体做对比，很形象了~

  • CPU=脑袋：每个人会作的事情都会一样(微指令集的差异)，但主要都是透过脑袋瓜子来进行判断不控制身体各部分的活动；
  • 内存储器=脑袋中的记录区块：在实际活动过程中，我们的脑袋瓜子能够将外界的互动暂时记录起来，提供CPU来迚行判断；
  • 硬盘=脑袋中的记忆区块：将重要的数据记录起来，以便未来将这些重要的经验再次的使用；
  • 主板=神经系统：好像人类的神经一样，将所有重要的组件连接起来，包括手脚的活动都是脑袋瓜子发布命令后， 透过神经(主板)传导给手脚来迚行活动啊！
  • 各项接口设备=人体不外界沟通的手、脚、皮肤、眼睛等：就好像手脚一般，是人体不外界互动的重要关键！
  • 显示适配器=脑袋中的影像：将来自眼睛的刺激转成影响后在脑袋中呈现，所以显示适配器所产生的数据源也是CPU控制的。
  • 电源供应器 (Power)=心脏：所有的组件要能运作得要有足够的电力供给！

3. 计算机上面常用的计算单位

  • 容量单位 １Byte=8 bits

  • 速度单位 CPU的频率：MHz、GHz >CPU的频率就是CPU每秒钟可以进行的工作次数，频率越高，表示CPU单位时间可以做更多的事。 >外频：CPU与外部组件进行数据传输时的速度 >倍频：CPU内部用力啊加速工作效能的倍数 >CPU频率=外频x倍频 >Tips:因为每个CPU的微指令集不相同，架构也不见得一样，每次频率能够进行的工作指令数也不同，所以，频率目前仅能用来比较同款CPU的速度！

网络传输方面：Ｍbps(Mbits per second)

4. 个人计算机架构与接口设备

两大主流PC架构如下：

前面的一幅采用两个网桥来控制各组件的沟通，分别是： 1. 北桥：负责连接速度较快的CPU、主存储器和显卡等 2. 南桥：负责连接素服较慢的周边接口，包括硬盘、USB、网络卡等。

而后一副图的不同之处在于主存储器直接与CPU沟通而不通过北桥。

总线

决定每次工作可以传送的数据的大小，北桥所支持的频率我仧称为前端总线速度(Front Side Bus, FSB)，每次传送的位数称为总线宽度，常见的总线宽度有32/64位。FBSX总线宽度就是每秒可传送带最大数据量。

内存

  • CPU所使用的数据都是来自主存储器(main memory)，不论是软件程序还是数据，都必须要读入主存储器后CPU才能利用。
  • 个人计算机的主存储器主要组件为动态随机存取内存(Dynamic Random Access Memory, DRAM)， 随机存取内存叧有在通电时扄能记弽不使用，断电后数据就消失了。
  • 双通道设计：传统的总线宽度一般大约仅达64位，为了要加大这个宽度，因此芯片组厂商就将两个主存储器汇整在一起， 如果一支内存可达64位，两支内存就可以达到128位了。
  • CPU与主存储器的外频应该要相同才好。
  • 除了主存储器之外，整个计算机当中还有许多内存！最为熟知的就是第二层高速缓存(L2 cache)因为它整合到CPU内部，因此这个L2内存的速度必须要CPU频率相同。使用DRAM是无法达到这个频率速度的，此时就需要静态随机存取内存(Static Random Access Memory, SRAM)的帮忙了
  • BIOS(Basic Input Output System)是系统在开机时候首先会去读取的一个小程序，它被写死在主板上的一个内存芯片中，这个内存芯片在没有通电时也能够将数据记录下来，是只读存储器(Read Only Memory, ROM)。

显卡

显卡对于图形影像的显示扮演关键的角色。 一般对于图形影像的显示重点在于分辨率与颜色深度，因为每个图像显示的颜色会占用掉内存， 因此显卡上面会有一个内存的容量，这个显卡内存容量将会影响到最终你的屏幕分辨率和颜色深度的。 除了显卡外，由于3D游戏和3D动画的流行，显卡的运算能力越来越重要，3D的运算早期是交给CPU运作的，后来显卡厂商直接在显卡上嵌入一个3D加速芯片，即GPU。

硬盘与存储设备

  • 传输接口： IDE接口：接口插槽所使用的扁平电缆较宽，每条扁平电缆上面可以接两个IDE装置 SATA接口：插槽要比IDE接口小很多，每条SATA连接线仅能接一个SATA装置，主流 SCSI接口：工作站等级以上的硬盘传输接口，PC上不常见。
  • 参数：容量、缓冲存储器、转数

PCI适配卡

如果用户有额外需要的功能卡， 就能够安装在这种PCI界面插槽上，现在主板上已经整合相当多的设备和组件了，包括声卡、网卡、USB控制卡、显卡等，所以现在主板上安插的PCI适配卡很少了。

主板

上面所谈到的所有组件都安插在主板上。

设备I/O地址和IRQ中断通信

I/O地址类似于每个装置的门牌号，一般两个装置不能使用同一个I/O地址，而IRQ可以想象成各个门牌连接到邮件中心（CPU）的专门路径。各个装置可以透过IRQ中断新到来告诉CPU该装置的工作情况，以方便CPU进行工作分配任务。

CMOS与BIOS

  • CMOS主要功能为记录主板上面的重要参数，包括系统时间、CPU电压与频率、各项设别的I/O地址与IRQ等，由与这些数据的记录要花费电力，因此主板上面有电池。
  • BIOS为写入到主板上某一块 flash或EEPROM 的程序，他可以在开机的时候执行，以加载CMOS中的参数，幵尝试呼叫储存装置中的开机程序，进一步进入操作系统中。BIOS程序也可以修改CMOS中的数据，每种主板呼吁BIOS设定程序的按键都不同，一般桌面计算机常见的是使用[del]按键进入BIOS设定画面。
